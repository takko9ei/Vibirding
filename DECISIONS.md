# DECISIONS — 取舍记录

> 每个取舍记三行：**决定 / 为什么 / 代价或取舍**。面试逐字稿用。

## read_log 只是"个人弱先验"，季节/分布核验另设权威工具
- **决定**：`read_log` 的职责收敛为"查我自己的历史观测"（个人记录 / 弱先验）；"某地某季是否合理出现某种鸟"这类权威核验，留给独立的鸟种分布工具 range_check（数据源 eBird，已升为正式切片 S3，见 §7/§10）。
- **为什么**：个人日志不代表生物地理事实——没记录过 ≠ 那儿没有，记录过也只是弱信号；权威物候/分布数据才是正主。v1 实测中 read_log 对季节判断几乎没提供权威价值，模型只能回退到自身知识。
- **代价/取舍**：v1 暂不接 eBird（控成本与范围），季节核验先靠模型知识 + read_log 弱先验；权威分布工具 range_check 已排为正式切片 S3；核验子 agent 仍作为后续进阶（§11）。
